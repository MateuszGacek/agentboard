# Project Brief

## Product name

**AgentBoard**

## Tagline

**AI Kanban for software teams shipping with agents.**

## One-liner

AgentBoard is a Kanban productivity platform for AI software agencies that helps teams clarify work, control WIP, prioritize delivery, and improve task quality with AI.

## Recruitment goal

This is a public GitHub and live-deployed recruitment project. It should prove that the developer can design, build, and deploy a complete product — not just implement isolated UI screens.

The project should show:

- product thinking,
- UX decisions,
- design system discipline,
- frontend architecture,
- backend/API/database modeling,
- AI integration,
- deployment ownership,
- performance awareness,
- clear documentation.

## Target audience

Primary user:

```txt
AI software agency / AI development team
```

Example personas:

- AI developer managing agent-based implementation tasks.
- Tech lead reviewing vague tasks before implementation.
- Project owner checking delivery health and bottlenecks.
- Recruiter reviewing the candidate's ability to ship product-quality software.

## Problem

AI-driven software teams often move fast, but their task quality can be weak:

- vague task descriptions,
- unclear acceptance criteria,
- too much WIP,
- unclear priority,
- no easy view of blocked/overdue work,
- weak visibility for non-technical stakeholders.

Generic Kanban boards track cards, but they do not help improve task quality or delivery focus.

## Product promise

AgentBoard helps teams answer:

1. What should we work on next?
2. What is overloaded?
3. What is blocked or late?
4. Is this task clear enough to implement?
5. Can AI improve this task before an engineer/agent starts coding?

## Differentiators

AgentBoard should not compete with Trello feature-by-feature. It should win as a focused, polished portfolio product.

Key differentiators:

- AI task improvement with review/apply flow,
- WIP limits and warnings,
- dashboard focused on delivery signals,
- premium SaaS UI,
- multilingual interface,
- mobile-friendly Kanban interactions,
- real backend and deployment.

## Success criteria

The project succeeds if a reviewer can:

- open the live URL,
- use the app without setup,
- understand the product in under 60 seconds,
- move tasks and see persistence,
- open task detail and see useful product depth,
- trigger AI improvement safely,
- inspect clean TypeScript code,
- understand architecture from README/docs,
- see that the product is deployed professionally.

## Product principles

### Clarity over complexity

Every screen should answer what the user can do next.

### Real flows over fake polish

A smaller working flow beats a large static UI.

### AI as workflow enhancement

AI should improve task quality, not become an unfocused chatbot.

### Mobile is a first-class review path

The app must work well on phone/tablet because reviewers may open it on mobile.

### Recruiter-friendly transparency

Known limitations should be documented honestly. Do not overclaim.
